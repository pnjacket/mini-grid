After the v1.7 root-finder optimization pass, the slowest built-in is FORECAST.ETS (~80µs/call over a 24-point window — ~40× the COUNTIF path — because Holt-Winters runs a smoothing-parameter grid-search per call). Each forecast cell FORECAST.ETS-forecasts the next point from the W cells above it in its own column (a deep vertical chain), plus a value-neutral + 0*B{row} term that injects an editable dependency at every row — so a single edit to B1 cascades the whole graph, while a B{last} edit stays a tiny sub-graph. Params: ?rows= (default 3000), ?cols= (default 4), ?window= (default 24), ?season= (default 4).
